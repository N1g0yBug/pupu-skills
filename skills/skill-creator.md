# skill-creator

## 触发条件
当现有技能无法覆盖新任务，或某类任务反复出现且步骤稳定时，使用此技能创建新的技能文档。

## 执行步骤
1. 判断是否该建技能：
   - 同类任务出现 3 次以上；
   - 执行步骤可标准化；
   - 结果可验证。
2. 定义技能边界：明确“做什么 / 不做什么”，防止技能过宽。
3. 设计触发词：
   - 中文 + 英文；
   - 任务词 + 工具词 + 结果词；
   - 控制在 6~12 个高质量触发词。
4. 使用统一 frontmatter：`name/description/triggers/calls`。
5. 正文结构固定：触发条件 -> 执行步骤 -> 代码参考 -> 注意事项 -> 改进记录。
6. 执行步骤必须可落地：每一步有动作、有输入、有产出。
7. 代码参考给“最小可运行示例”，避免只给伪概念。
8. 完成后做自测：
   - 用触发词模拟路由；
   - 检查是否会误触发其他技能；
   - 检查是否包含禁用技术（例如 Python）。
9. 迭代机制：每次失败案例沉淀到“注意事项/改进记录”。

## 注意事项
- 技能是“操作手册”，不是概念文章；必须让执行者可直接照做。
- description 要写“结果价值”，不要写成“使用某某工具”。
- 避免把多个不相关流程塞进一个技能。
- 每个技能都应包含失败处理建议。
